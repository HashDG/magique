La hiérarchie d'agents Magique


Cliquez sur les différents agents pour obtenir une description détaillée


 
 
  • Task : représente l'élément de calcul de l'application.
  • TaskFactory : a pour role de générer les objets Task.
  • TaskManager : algorithme applicatif (vision très globale du calcul).
  • Administrator : outil d'administration et de monitoring.
  • TaskDispatcher : stocke les tâches pour les distribuer sur les différentes plateformes.
  • PlatformBoss : gestion de "groupe" pour les actions de broadcast ...
  • PlatformManager : gestion des ressources (en terme d'agent et de calcul) d'une plateforme Magique.
  • RepositoriesManager : gestionnaire des dépôts de tâches. Son rôle est de maintenir une base de donnée répartie fonctionnant avec un système de mirroirs permettant d'enregistrer les résultats des différentes tâches. Il créé dynamiquement les différents ResultRepository et les attribue aux différentes plateformes.
  • ResultRepository : un entrepôt pour les résultats des tâches terminées. Son rôle consiste à fournir un moyen de stockage aux tâches qui sont finies et de permettre la consultation des résultats des autres tâches.



Task.

La tâche est l'élément de calcul de l'application pouvant être distribuée. A une tâche est associé un ReckonerAgent qui assure l'intégration de la tâche dans le framework (la tâche est un objet et non un agent). Ceci permet à la tâche d'avoir accés aux possibilités de communication du framework (appel de compétences) notamment pour connaître le résultat d'autres tâches ou pour communiquer avec d'autres tâches. Chaque tâche à un cycle de vie précisé par le schéma suivant:


Cycle de vie d'une tâche


TaskFactory.

Les TaskFactory générent les tâches à la demande du TaskManager.


TaskManager.

Cet agent, spécifique à l'application, peut être vu comme l'implantation de l'algorithme. En s'appuyant sur les TaskFactory, il génére les objets de type Task qu'il envoie au TaskDispatcher.


TaskDispatcher.

Les tâches générées par le TaskManager sont déposées dans le TaskDispatcher. Celui-ci les stocke et les distribue aux différents PlatformManager qui les demandent. La première implantation de Rage faisait du TaskDispatcher l'initiateur de la distribution des tâches. Plusieurs constatations amenérent une modification du mécanisme de distribution: les explications complètes sont données ici.
De plus, le TaskDispatcher assure une partie du processus de tolérance aux pannes. Il gére un timeout sur chaque tâche distribuée ce qui lui permet de vérifier périodiquement les tâches et de relancer celles dont le délai de garde a expiré.


RepositoriesManager et ResultRepository.


PlatformManager.


ReckonerAgent.

Ces agents sont les entités responsables des calculs. Ils reçoivent les tâches des PlatformManager, exécutent le calcul puis envoient le résultat vers un ResultRepository. De plus, il permettent aux tâches de bénéficier du support de communication du système multi-agents.
